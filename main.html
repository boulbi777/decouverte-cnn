def build_CNN(input_dim, output_dim, lr=0.001):
    
    # Initialisation du réseau
    model  = Sequential()
    
    #Différentes couches
    model.add(Conv2D(filters=30, kernel_size=(3,3), input_shape=input_dim))
    model.add(Activation(activation="relu"))
    model.add(MaxPooling2D(pool_size=(2, 2)))
    model.add(Dropout(rate=0.1))

    #-----------------------------------------------------------------------------------#

    model.add(Conv2D(filters=60, kernel_size=(3,3)))
    model.add(Activation(activation="relu"))
    model.add(MaxPooling2D(pool_size=(2, 2)))
    model.add(Dropout(rate=0.1))

    #----------------------------------------------------------------------------------#

    model.add(Conv2D(filters=120, kernel_size=(3,3)))
    model.add(Activation(activation="relu"))
    model.add(MaxPooling2D(pool_size=(2, 2)))
    model.add(Dropout(rate=0.1))

    #----------------------------------------------------------------------------------#

    model.add(Flatten())
    model.add(Dense(units=150))
    model.add(Activation(activation="relu"))
    model.add(Dropout(rate=0.3))

    #----------------------------------------------------------------------------------#

    model.add(Dense(units=output_dim))
    model.add(Activation(activation="softmax"))    
    
    # initialiser l'optimiseur RMSprop
    opt = rmsprop(learning_rate=lr, decay=1e-6)
    
    #Compilation du modèle
    model.compile(loss='categorical_crossentropy', optimizer=opt, metrics=['accuracy'])
    
    return model

3. A quoi sert le DropOut ?¶

Dans la définition de la fonction, plusieurs couches sont définies les unes après les autres. Sur chaque couche, on précise le type de convultion souhaité (2D dans notre cas), la fonction d'activation, etc...

Le droupOut dans un réseau de neurone sert essentiellement à éviter de l’overfitting lors de l'entraînement du réseau. En effet, dans les CNN, le passage plusieurs fois sur les mêmes pixels d'une image donnée (principe de la convolution) peut rapidement conduire à ce phénomène de surapprentissge où les neurones des couches cachées (ou intermédiaire) ajustent leurs poids qui se collent trop aux données d'entraînement. Le dropOut est donné avec un pourcentage qui détermine la proportion de neurones à tuer à la sortie de chaque couche cachée où elle est spécifiée (c'est le paramètre 'rate' dans la fonction), et ceci, afin d'éviter une surconcentration d'informations. Tuer un neurone reviend simplement à fixer son poids à 0. Ainsi, le dropOut a pour objectif de permettre une bonne capacité de généralisation du réseau de neurones construit afin d'améliorer sa capacité de prédiction au vu de nouvelles images. Cette a déjà fait ses preuves dans le monde du deep learning. Les proportions de neurones à tuer varient généralement entre 20% et 50%.

Nous utilisons la classe ImageDataGenerator de la librairie keras qui est très fluide et efficace pous ces types d'opérations. 2. Phase d'entrainement¶

Un paramètre important lors de l'entrainement de notre réseau est le nombre d'étapes à faire par epoch. On dit qu'un réseau de neurone a effectué 1 epoch lorsqu'il a utilisé exactement 1 fois la totalité des observations (ou images) disponible dans la base d'entrainement. Puisque le batch_size spécifie le nombre d'images utilisé par étape d'entrainement, nous en déduisons qu'il faut $\frac{N}{BS}$ étapes pour atteindre 1 epoch, où $N$ est la taille de l'échantillon d'entrainement et $BS$ le batch_size. Nous allons spécifier ce paramètre maintenant, on le nommera nb_steps_per_epoch.

Q3. Faire de la Data Augmentation¶

La Data Augmentation est un procédé qui permet de modifier ou d'accroître les exemples d'entrainement de notre CNN afin que le réseau de neurones puisse explorer d'autres exemples et augmenter sa capacité de généralisation. Elle consiste à apporter des modifications à des images (zoom, rotation, etc...) afin de multiplier les images d'entrainement et de validation. Cette technique donne souvent de bons résultats pour combattre le surapprentissage. Nous allons toujours utiliser la classe ImageDataGenerator de keras en lui apportant simplement des modifications avec les paramètres de Data Augmentation que nous allons mettre en place. Il est extrêmement important de noter que la méthode de Data Augmentation implémentée dans keras ne consiste pas à multiplier les images d'entrainement mais applique à chaque image des modifications aléatoires choisies en laissant constant le nombre d'images dans les bases d'entrainement et validation (/!\).

Dans notre cas, nous allons appliquer certaines modifications. Voici quelques points notables :

  • Zoomer ne nous servira pas à grand chose pous ces types d'images puisqu'elles sont déjà de très faibles qualités.
  • Nous appliquons un déplacement aléatoire horizontal et/ou vertical de 20% au maximum.
  • Certaines images sont carrément retournés sur leur tête (horizontal_flip)
  • Le type de remplissage se fait par la méthode du voisin le plus proche selon la direction du décalage de l'image (paramètre fill_mode).

Au vu du modèle chargé, nous jugeons adéquat d'enlever les deux dernières couches (Dense et Activation) pour les remplacer par celles qui conviennent mieux.

La méthode pop() de kears pour enlever la dernière couche d'un réseau de eurones ne fonctionne pas sur la configuration tensorflow de notre PC. Nous avons dû définir une fonction drop_last qui le fait manuellement.

In [62]:
def drop_last(model):
    name_last_layer = str(model.layers[-1])

    model2 = Sequential()
    for layer in model.layers:
        if str(layer) != name_last_layer:
            model2.add(layer)
    return model2

Nous allons finalement définir une nouvelle fonction transfert_learnig_CNN qui nous permettra d'appliquer d transfert learning à partir d'un réseau préentrainé donné en paramètre. Il faut ensuite spécifier le nombre de dernière couches à enlever et la dimension de sortie. Nous utilisons pluôt la classe Model de keras au lieu de Sequentiel pour couper les dernières couches et rajouter le nôtre.

def transfert_learnig_CNN(pretrained_model, nb_last_layers_to_drop, output_dim):
    
    #couper autant de fois la dernière couche demandée
    for _ in range(nb_last_layers_to_drop):
        pretrained_model = drop_last(pretrained_model)
    
    #Empilement des modèles
    x = Dense(units=50)(pretrained_model.get_output_at(-1))
    x = Dense(units=output_dim)(x)
    o = Activation(activation="softmax")(x)
    
    opt = rmsprop(learning_rate=0.0001, decay=1e-6)
    model2 = Model(inputs=pretrained_model.input, outputs=[o])

    return model2
